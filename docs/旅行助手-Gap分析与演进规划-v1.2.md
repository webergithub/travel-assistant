> 类型：Gap 分析与版本化演进规划（三部曲第三份）。文档版本 v1.2（2026-07-23 E1 收口后状态刷新）。

> 一句话：Gap = 规划 PR − 现状 FR，逐条带证据、定级、给破坏性关闭验收，映射到 v0.3–v1.0。

## 0. 文档定位

| 文档 | 回答 | 编号 |
|---|---|---|
| ① 需求规格说明书 | 现在是什么 | FR / KI |
| ② 需求规划文档 | 应该是什么 | PR |
| ③ 本文 | 差距在哪、怎么补、什么次序 | G-域-x |

输入：①② + 附录 B 代码核实清单（本文全部现状陈述的出处）。输出：分域 Gap 清单、优先级矩阵、版本化演进路线、三向追溯矩阵。

## 1. 分析方法

每条 Gap 五元组：现状（带证据编号 V-x）/ 目标（引 PR）/ 差距陈述 / 级别 / 关闭验收（破坏性，写在 §5 各阶段表）。**Gap 只在破坏性验收通过后关闭，代码合入不算。**

| 级别 | 含义 | 判据 |
|---|---|---|
| L1 品类硬伤 | 叫不叫得上「行程规划工具」 | 品类核心承诺不成立 |
| L2 任务阻断 | 核心任务走不通 | 成功条件缺前提 |
| L3 健壮性 | 静默失败/数据风险/安全兜底 | 失效不可见或恢复不了 |
| L4 体验 | 可用但别扭 | 有绕行路径 |
| L5 工程债 | 用户不可见，拖慢演进 | 不可测/残留/返工率 |

## 6. 依赖与风险

| 风险 | 影响 | 缓解 |
|---|---|---|
| 免费地图数据源政策变化（Nominatim 限速收紧/Carto 停免费） | 搜索与底图两条腿同时受伤 | G-MAP-1 的多源切换架构同时预留商用源接口；缓存已降依赖 |
| 大陆网络对 OSM 系资源的可达性波动 | 核心体验环境性劣化 | 同上；天地图/高德接入评估提前到 E1 |
| 平台不配 ANTHROPIC_API_KEY 的现状持续 | AI 叙事只对自带 key 用户成立 | E0 的 mock 演示模式兜底产品叙事；配 key 前先落 G-OPS-3 配额 |
| SQLite 并发写上限 | 协作(E3)后写冲突概率上升 | E3 前评估 Postgres 迁移（C-5 预留）；备份先行使迁移可演练 |
| 跨条目联动：游客迁移(G-DATA-2)依赖账号合并语义，与限流(G-OPS-3)的 guest 清理策略互相约束 | 清理误删待迁移游客 | 清理只针对「零行程且 30 天未活跃」游客 |
| 单人维护带宽 | 阶段拖长 | 每阶段量级已控制在 1–2 轮迭代；出口标准不缩水，宁可顺延 |
